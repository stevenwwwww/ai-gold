# 下一步操作 — 只需完成 3 件事

你已经生成了 SSH 密钥，接下来按顺序完成下面 3 步即可。

---

## 第一步：在 ECS 上添加公钥（约 2 分钟）

1. 打开阿里云控制台 → ECS → 你的实例 → **远程连接**
2. 点击 **Workbench 立即登录**（无需密码，浏览器直接打开终端）
3. 在终端里依次复制粘贴执行：

```bash
mkdir -p ~/.ssh && chmod 700 ~/.ssh
echo "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIMzX2AjdqP1CsWcVeHx92Duhn/GqPn5Sn9O0spQg6ENp github-actions-deploy" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
echo "公钥已添加"
```

完成。不要关闭 Workbench，继续下一步。

---

## 第二步：在 ECS 上完成首次部署（约 15–20 分钟）

在同一个 Workbench 终端里执行（**私有仓库需把 YOUR_TOKEN 换成你的 GitHub Personal Access Token**）：

```bash
export REPO_URL="https://stevenwwwww:YOUR_TOKEN@github.com/stevenwwwww/ai-gold.git"
cd /opt && sudo mkdir -p doctor-saas && sudo chown $USER:$USER doctor-saas
git clone "$REPO_URL" doctor-saas
cd doctor-saas/deploy && bash bootstrap-workbench.sh
```

- 主应用来自 **stevenwwwww/ai-gold**，RAGFlow 会在脚本中自动从官方 `infiniflow/ragflow` 克隆（你无需单独拉取）
- **获取 Token**：GitHub → 头像 → Settings → Developer settings → Personal access tokens → Generate new token，勾选 `repo` 权限

脚本运行时会提示你粘贴公钥 —— **直接回车跳过**（第一步已经添加过了）。

脚本还会问你 `.env.production` 的配置：

- **QWEN_API_KEY**：必须填（通义千问 API Key，去 https://dashscope.aliyun.com/ 获取）
- **RAGFLOW_API_KEY**：可以留空，首次部署会先启动 RAGFlow，等部署完去 `http://8.140.221.86:8080` 注册并获取，再执行一次：

```bash
cd /opt/doctor-saas/deploy && sudo ./deploy.sh --skip-deps
```

---

## 第三步：在 GitHub 添加 Secrets（约 1 分钟）

1. 打开你的 GitHub 仓库 → **Settings** → **Secrets and variables** → **Actions**
2. 点击 **New repository secret**，依次添加：

| 名称 | 值 |
|-----|---|
| `ECS_HOST` | `8.140.221.86` |
| `ECS_USER` | `root` |
| `SSH_PRIVATE_KEY` | 打开本机 `~/.ssh/aliyun-deploy` 文件，**完整复制**全部内容（包括 `-----BEGIN` 和 `-----END` 两行） |

---

## 完成之后

以后每次执行 `git push origin main`，GitHub 会自动部署到你的 ECS。

**重要安全提示**：
- 若曾在文档或聊天中粘贴过 GitHub Token，请立即在 GitHub 删除并重新生成
- 若曾在聊天中粘贴过 SSH 私钥，建议重新生成密钥并更新 ECS 公钥和 GitHub Secrets：
```bash
ssh-keygen -t ed25519 -f ~/.ssh/aliyun-deploy -N "" -C "github-actions-deploy"
```
